# 支援職パーティーRPG 仕様まとめ

## コンセプト
主人公は支援専任(ヒール系スキルのみ)、他パーティーメンバーは自動戦闘。
プレイヤーはパーティー運営(雇用・維持費)と、支援スキルの操作(タップ指定→詠唱→発動)で戦況をコントロールする。

---

## MVP範囲(最初に実装する部分)

### マップ生成
- **自動生成ダンジョン**(BSP法: 二分空間分割で部屋を配置)
- 部屋タイプ(雑魚部屋/中ボス部屋/ボス部屋)をグラフで順序管理し、攻略の型を保証
- 部屋サイズにバリエーションを持たせ、**狭い通路**と**広い部屋**の両方を生成
- ステージ: 草原のみ、1ステージ
- 探索形式: フィールド探索型(部屋クリア制ではなく、シームレスに進む)

### 空間タイプ別の難易度メリハリ(意図的な設計)
| 空間タイプ | 特徴 |
|---|---|
| 狭い通路 | 避け場がない、緊張感重視。詠唱中の被弾リスクが高い |
| 広い部屋 | 立ち回り重視。範囲ヒールや位置取りが活きる(ヴァンサバ風) |

### 敵配置
- **固定配置**: 中ボス・ボスなど、生成時に位置決定
- **ランダムポップ**: 雑魚、プレイヤーが近づくと出現

### 主人公操作
- 左スティック: 移動
- 単体ヒール: 左上パーティー一覧 or 画面上キャラ直接タップで対象指定 → 詠唱 → 発動
- 詠唱中は低速移動(足は止まらない)
- 威力/詠唱の設計(将来のスキル追加を見据えた基準):
  - 単体ヒール: 威力高、詠唱中
  - 範囲指定ヒール(タップで位置指定): 威力中、詠唱長
  - グループヒール(自動全体対象): 威力低、詠唱なし(即時、保険用)

### パーティー・経済
- 酒場で仲間を雇用(雇用費+維持費)
- MVPで雇用可能な職業: 戦士(前衛)、弓使い(後衛)
- 維持費はステージ開始時に徴収、不足すると脱退
- 維持費システムはMVPから導入

### ロストシステム(設計のみ、実装は後回し)
主人公死亡 = パーティー全滅として扱う。
| 項目 | 扱い |
|---|---|
| 装備 | ロスト |
| 所持金 | ロスト |
| 仲間全員 | ロスト |
| 累計経験値/レベル | 維持 |
| そのステージ内の獲得経験値 | ロスト(ステージクリア後は対象外) |

緩和策: ビルドの一部到達点で「ロスト無効化」を習得可能(後述のビルド拡張時に実装)。
→ データ構造(装備・所持金・仲間・経験値の管理)は、後からロスト処理を差し込みやすいよう分離しておく。

## 将来拡張(MVP後に実装予定)

### 世界観・ステージ
- ワールドマップから選択して入るダンジョン: 草原、森、荒野、洞窟、墓地
- 各エリアにステージ3つ程度(雑魚→中ボス→ボス)

### 雇用職業(拡張)
- 戦闘職(前衛/後衛): 戦士、重戦士、魔法使い、盗賊、弓使いなど
- 一般職(用途不明、隠し効果の伏線): 主婦、老人、商人、農民、酔っ払いなど
  - 特定ビルド/職業と組んだ時のみ発動する隠し効果を想定

### 主人公の成長
- 敵撃破で経験値取得→レベルアップ→スキル習得(スキルツリー)
- ボス撃破等でビルドポイント取得

### 5大ビルド(スキルツリーとは別軸、大方針)
1. **回復型** — ヒール特化
2. **バフ型** — 加速方向(移動・攻撃速度up、詠唱中の速度低下軽減、将来的に無詠唱化の到達点)
3. **デバフ型** — 減速方向(足止め、行動阻害)
4. **軍師型**(経済+人材統合) — 維持費軽減、雇用費割引、雇用枠+、一般職の隠しスキル開放、脱退リスク軽減
5. **召喚型**(精霊召喚) — 単体強力精霊/数の暴力精霊/継続バフ精霊の3パターン

設計方針: ビルド同士の相性を内部では設計しつつ、ゲーム内で"最適解"を明示しすぎない。
パラメータは掛け算・閾値効果を活用し、一見弱い組み合わせにも化ける余地を残す(プレイヤーが想定外の組み合わせを発見できるように)。

### 職業(ビルドとは別軸、Lv10で選択、3段階昇格)
| 職業 | 特徴 | 相性の良いビルド |
|---|---|---|
| 回復系 | 特化型、回復力・詠唱短縮・無詠唱化を極める | 回復型、バフ型 |
| 賢者系 | 汎用型、バフ・デバフ・回復を平均的にこなす | どれとも組みやすい |
| 軍師系 | 指揮特化、パーティ運用(維持費・雇用枠・一般職性能)を伸ばす | 軍師型、召喚型 |

### 敵バランス方針
- 味方(仲間+精霊)は物量、敵は少数精鋭で強化
- 雑魚ステージでも個々にHPや攻撃パターンを持たせ、単純作業感を避ける
- 物量ビルド(召喚型/軍師型)が本領発揮できる中型敵ラッシュのステージも用意

### ダンジョンタイプ(将来的に2種類)
- ヴァンサバ風: 広い空間で耐える(MVPでは「広い部屋」として統合済み)
- ダンジョン型: 通路を進む(MVPのベースがこちら)

## 技術スタック
- Flutter + Flame(2Dゲームエンジン)
- 既存スタック(Capacitor, Claude Code, VS Code)との親和性を優先
- 物量戦でパフォーマンス限界に達した場合、Unity移行を段階的に検討

## 開発の進め方(Claude Code活用)
1. この仕様書をCLAUDE.mdとしてプロジェクトに配置
2. 実装はステップを区切って依頼する(一度に全部頼まない)
   - 例: 移動→単体ヒール(詠唱込み)→戦士配置→敵スポーン→マップ生成(BSP)→酒場UI→維持費
3. 各ステップごとに実機/エミュレータで動作確認し、体感ベースでフィードバック
